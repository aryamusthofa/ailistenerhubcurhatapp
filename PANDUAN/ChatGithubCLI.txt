   1. Firebase Not Initialized: Error [core/no-app] No Firebase App '[DEFAULT]' has been created muncul karena AuthNotifier (yang ada di dalam
      authProvider) langsung mencoba memanggil FirebaseAuth.instance saat aplikasi baru saja dibangun, padahal Firebase.initializeApp() belum selesai

  Solusi Cepat & Tepat:

  initializer jika kita belum yakin Firebase sudah siap.

  Apalagi di Linux (yang Firebase-nya kita mock/skip), kode ini pasti crash kalau mencoba akses FirebaseAuth.instance.

  Saya akan perbaiki lib/presentation/providers/auth/auth_provider.dart sekarang juga agar aman di Linux maupun platform lain.


  Rencana Perbaikan:
   1. Ubah _auth menjadi FirebaseAuth? (nullable).
   2. Hapus inisialisasi langsung FirebaseAuth.instance.
   3. Di dalam constructor, cek platform dulu. Jika Linux, biarkan null. Jika bukan, baru coba ambil instance (dengan try-catch).

  ðŸ”¥ Cara Menyambungkan Firebase (Agar Login Berfungsi)

  Saat ini, Firebase gagal jalan di Linux karena Firebase official plugin (FlutterFire) memang TIDAK MENDUKUNG Linux secara native. Dia cuma support
  Android, iOS, Web, dan macOS.


  Namun, Anda punya 3 Opsi untuk membuat backend-nya nyambung:


  Opsi 1: Pindah ke Android (Recommended untuk Produksi)
  Ini cara paling standar dan stabil.
   1. Siapkan Emulator/HP Android: Colok HP Android Anda via USB dan aktifkan USB Debugging.
   2. Jalankan di Android:
   1     flutter run -d <id_device_android>
   3. Setup Firebase: Anda perlu mendownload file google-services.json dari Firebase Console dan menaruhnya di android/app/.
       * Apakah Anda sudah punya file ini? Kalau belum, saya bisa pandu cara dapatnya.


  Opsi 2: Gunakan Firebase Web (Bisa jalan di Linux via Chrome)
  Jika Anda ingin tetap develop di laptop tanpa emulator berat, cara paling "sakti" adalah menjalankan versi Web.
   1. Jalankan Mode Web:
   1     flutter run -d chrome
   2. Kelebihan: Firebase Web didukung penuh! Login Google, Firestore, Auth, semua jalan mulus di browser Chrome di laptop Linux Anda.
   3. Setup: Anda perlu menambahkan konfigurasi Firebase Web di main.dart (API Key, App ID, dll) saat Firebase.initializeApp.


  Opsi 3: Auth Mocking (Untuk Development Linux Desktop)
  Jika Anda kekeuh ingin pakai aplikasi desktop Linux native (bukan Web/Android) tapi tetap ingin bisa "pura-pura login" agar bisa masuk ke menu Chat:
   * Kita bisa memodifikasi AuthNotifier lagi.
   * Jika di Linux, tombol "Login Google" akan langsung bypass ke Home Screen (Login sebagai "Guest User" lokal).
   * Data chat disimpan di Database Lokal (Hive/SQLite) saja, tidak disinkron ke Cloud.
